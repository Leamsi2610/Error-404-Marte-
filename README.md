# Error-404-Marte- Hecho por Sara Fisac, Alonso Gallego, Ismael Haro, Ana Pérez y Laura Suárez  [https://leamsi2610.github.io/Error-404-Marte-/404.html](https://leamsi2610.github.io/Error-404-Marte-/t)
# De qué va nuestro proyecto
Nuestro objetivo era crear una página de Error 404 que tuviera personalidad y que además encajara con una temática espacial divertida. La idea era convertir la página en una pequeña escena en la que un astronauta acaba en Marte por error, haciendo referencia al error 404. Así, si alguien se pierde en la web, al menos aterriza en un lugar curioso: un “Marte improvisado” lleno de elementos flotantes, colores suaves y un mensaje simpático.
# Cómo nos organizamos como equipo
Nos repartimos las tareas según lo que mejor sabía hacer cada uno, aunque la mayor parte del tiempo los roles no eran fijos y nos dábamos apoyo unos a otros en diferentes tareas o en aquello que fuese surgiendo. Después de hacer un brainstorming y bocetar algunas ideas, escogimos entre todos el diseño final. Se podría decir que la forma de trabajo se dividía en dos partes principalmente: la parte de código y la parte gráfica/visual que hicimos con Adobe Illustrator. En todo momento estuvimos colaborando en ambas partes, dependiendo de qué hiciese falta.
# La parte creativa y visual
Antes de empezar con el código, pensamos en qué estilo queríamos. Nos decidimos por un estilo flat design con degradados para los elementos flotantes (rocas, un alien, un astronauta y Marte con el cohete), una tipografía futurista (Share Tech) y un fondo oscuro para simular el espacio. Queríamos que el usuario sintiera que había entrado en un rincón extraño y simpático. La elección de cada imagen y su posición inicial dentro de la página fue parte importante del diseño, porque la idea era lograr una escena equilibrada pero con movimiento.
# Construyendo la estructura en HTML
El HTML lo mantuvimos lo más limpio posible, organizando los elementos en contenedores específicos para poder animarlos más fácilmente después. Cada imagen tiene su propio espacio y sus coordenadas iniciales, lo que nos permitió tener control sobre dónde aparece todo sin que se descoloque cuando aplicamos animaciones.
# Dándole estilo y movimiento con CSS
Aquí fue donde más experimentamos. Usamos clamp() para que los textos se ajustaran a cualquier pantalla, definimos variables CSS para las posiciones y aplicamos transiciones suaves para que los objetos flotaran en vez de “saltar”. También jugamos con transform y con el z-index para que nada tapara el texto principal. Uno de los retos fue coordinar todos los elementos flotando a la vez sin que quedara caótico, pero al final logramos un equilibrio bastante resultón.
# La animación con JavaScript
Para dar el toque final, usamos JavaScript para animar las estrellas y mover algunos elementos cuando el usuario mueve el ratón. Esto ayudó a que la página pareciera más viva, como si realmente estuvieras sobrevolando un paisaje espacial. Ajustar el rendimiento fue un punto importante, porque demasiadas animaciones pueden ralentizar la página, así que optamos por usar transformaciones que consumen menos recursos.
# Los retos que nos encontramos
No todo salió a la primera. Tuvimos que ajustar muchas veces el tamaño de las imágenes, sus posiciones, el comportamiento al mover el ratón e incluso el balance entre lo visual y lo funcional. También aprendimos a coordinarnos mejor como equipo, especialmente cuando varios estábamos tocando partes que se relacionaban entre sí. Finalmente, no conseguimos que la web fuese responsive tras probarlo en repetidas ocasiones :(
# Lo que aprendimos
Más allá del resultado final, este proyecto nos enseñó a trabajar juntos, a experimentar con animaciones y a pulir detalles para que la página fuera bonita y fluida. Cada uno aportó algo distinto, y gracias a eso conseguimos una página 404 que no solo cumple su función, sino que también muestra la creatividad del grupo y nuestras ganas de hacer algo diferente.
